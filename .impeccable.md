## 设计上下文

### 用户

平台服务于三个重叠的用户群体：

1. **有经验的 Java 开发者，初次接触 AI** — 熟悉 Spring Boot 和企业级 Java 的资深工程师，首次探索 AI Agent 工程。他们期望以代码为中心、技术性强的内容，复杂度循序渐进。他们想理解*为什么*，而不仅仅是*怎么做*。
2. **学生和自学者** — 不同水平的开发者，通过动手编码学习 AI Agent 模式。他们需要清晰的可视化、逐步动画和循序渐进的结构化学习路径。
3. **采用 AI 工具的团队** — 正在评估或将 AI 编码 Agent 集成到工作流中的工程团队。他们需要架构理解和可落地的实践模式。

**共同背景**：用户带着开发者的期望来到这里 — 他们想读代码、看运行结果、建立心智模型。他们经常在学习平台和自己的 IDE/终端之间切换。

### 品牌个性

**清晰、渐进、技术感**

- **清晰** — 每个元素都能即时传达其用途。没有装饰性噪音。标签精准。导航直观。可视化用于解释，而非装饰。
- **渐进** — 界面反映学习旅程：概念层层递进，可视化随课程演进，UI 奖励更深入的探索。前进感始终可见。
- **技术感** — 美学风格使用开发者的语言。等宽代码字体突出。深色模式为主。信息密度受到尊重。UI 像开发者工具，不像营销网站。

### 美学方向

**参考美学：开发者工具（VS Code、Claude Code CLI）**

界面应该像一个精心打造的开发者工具 — 不是文档网站，不是课程平台，而是开发者会自然地保持在其 IDE 旁边打开的东西。

**视觉基调**：
- 深色模式作为主要身份。深 zinc 色调 (`zinc-950` 基底) 是标志性外观。
- 代码优先：等宽字体占主导，语法高亮突出，代码块是一等公民。
- 信息密集但不杂乱。充裕的间距分隔逻辑区块。
- 5 层颜色系统 (blue/emerald/purple/amber/red) 为全部 12 节课提供视觉脚手架 — 一致但不单调。

**反面参考**：
- 不是带有首屏动画和渐变背景的炫酷营销网站
- 不是带有进度条、徽章和游戏化装饰的传统 LMS
- 不是随意的博客布局

### 设计原则

1. **代码即内容** — 代码块、diff 和语法高亮源码是主要内容类型，不次于散文。围绕代码优先阅读设计：深色背景、清晰的语言标签、充裕的行高、复制功能。

2. **可视化服务教学** — SVG 图表、流程图和逐步动画的存在是为了构建心智模型。它们必须准确、清晰且可交互。动画服务于理解，而非装饰。

3. **尊重开发者习惯** — 深色模式、等宽字体、键盘友好导航、信息密度。UI 应该让常驻 VS Code 和终端的人感到自然。不要把学习平台的惯例强加给开发者用户。

4. **渐进式披露，渐进式复杂度** — 12 节课程结构对应递增的复杂度。UI 应该让这种递进可见（时间线、层级徽章），但不要用后续才引入的概念压倒前面的课程。

5. **默认多语言** — 中文是主要语言，同时支持英文和日文。所有设计决策必须在三个语言环境下都能工作：文本长度差异、CJK 排版和双向布局考虑。

### 现有设计系统参考

**技术栈**：Next.js 16 + React 19 + Tailwind CSS v4 + Framer Motion, TypeScript 5

**颜色系统**：
- 中性色：Zinc 色阶 (50-950)，深色模式主色
- 强调层：Blue `#3B82F6`（工具）、Emerald `#10B981`（规划）、Purple `#8B5CF6`（记忆）、Amber `#F59E0B`（并发）、Red `#EF4444`（协作）
- 聊天：Blue-600 用户气泡，zinc 助手气泡

**排版**：系统字体（Geist via next/font），等宽字体栈用于代码

**组件**：手写（无组件库）— Card、Badge、Tabs、Sidebar、Header

**动画**：Framer Motion — 入场动画 (opacity + y)、SVG 过渡 (0.4s)、滚动触发、逐步回放
